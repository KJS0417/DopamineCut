# 5. 테스트 방안
본 프로젝트('DopamineCut')는 사용자 기기 화면에서 발생하는 이벤트를 백그라운드에서 실시간으로 감지하며, AI로 분류하여 서버로 데이터를 동기화하는 복잡한 처리 과정을 가지고 있다. 시스템 성능 및 신뢰성 검증을 위하여 단위 테스트와 통합 테스트를 아래와 같이 수행한다.

## 5.1. 단위 테스트
단위 테스트는 시스템을 구성하는 개별 모듈(메소드, 함수, 클래스 등)의 의도한 기능이 독립적으로 잘 작동되는지 검증한다. 핵심이 되는 기능은 OCR을 이용한 텍스트 정제와 AI를 이용한 카테고리 분류 모듈을 집중적으로 테스트한다.

### 5.1.1. OCR 텍스트 추출 및 정제의 정확성을 위한 모듈 테스트 (`OCRProcessor`)
**테스트 목적**: 다양한 해상도와 UI의 단어와 불필요한 공백 등의 노이즈가 섞인 화면에서 의미 있는 텍스트만 정확하고 빠르게 추출하는지 검증한다.
**테스트 방법**:
  * 유튜브 쇼츠, 인스타그램 릴스 등 UI 구조가 다른 이미지를 캡처하여 50장 가량 준비한다.
  * `extractTextFromScreen()` 함수에 이미지를 입력한 후 반환되는 RawText의 오타율을 측정한다. (응답 시간 목표 : 0.5초 이내)
  * 추출된 RawText를 `refineText()` 함수에 입력하여, 특수문자, UI의 단어, 불필요한 공백 등이 성공적으로 필터링되는지 검증한다.

### 5.2.2. AI를 이용한 카테고리 분류의 정확성을 위한 모듈 테스트 (`GeminiCategoryClassifier`)
**테스트 목적**: 정제된 텍스트 데이터와 프롬프트를 결합하여 Gemini API에 전송했을 때, 지정된 카테고리 중 하나로 정확하게 맵핑되는지 검증한다.
**테스트 방법**:
  * 게임, 유머, 교육, 사회 등 카테고리가 명확한 정제 텍스트 샘플을 50건 가량 구성하여 `requestToGemini()` 함수에 입력한다.
  * 반환된 카테고리 결과값이 사람이 미리 분류해 둔 카테고리 텍스트와 85% 이상 일치하는지 확인한다.
  * `applyDopamineScore()` 함수에 특정 유저의 가중치 데이터(`restriction_weights`)와 카테고리를 입력하여, 사전에 정의된 벌점(도파민 점수 차감 점수)이 정확하게 연산되는지 확인한다.

## 5.2. 통합 테스트
통합 테스트는 앱 실행부터 웹 커뮤니티의 데이터 반영까지 여러 컴포넌트나 모듈이 함께 동작하는지 확인을 하기 위한 테스트로, 전체적인 데이터 흐름과 시스템 동작 과정을 검증한다. 에뮬레이터 환경에서는 Accessibility API의 동작이 상이할 수 있으므로, **실제 안드로이드 기기** 환경에서 수행한다.

### 5.2.1. 파이프라인(처리 과정)의 전체적인 동작 흐름 검증 (End-to-End Test)
**테스트 시나리오**:
  * **영상 진입 감지**: 사용자가 기기에서 유튜브 앱을 실행하고 '쇼츠' 구역으로 이동한다. `AppManagerInterface`가 해당 앱의 UI를 정상적으로 감지하는지 확인한다.
  * **시청 의도 검증 및 캡처**: 숏폼 시청 시간이 3~5초 초과했을 때 `ViewTracker`가 동작하고, 시스템이 화면을 캡처하는지 검증한다.
  * **OCR 추출 및 AI 분류**: 캡처된 화면이 `OCRProcessor`와 `GeminiCategoryClassifier`를 거쳐 특정 카테고리('교육', '먹방' 등)로 판별되는지 로그를 확인한다.
  * **도파민 점수 적용 및 제어**: 판별된 카테고리에 따라 유저의 `daily_score`(금일 잔여 도파민 점수)가 올바르게 차감되며, 목표 횟수 초과 시 `AppBlockService`가 경고 또는 앱 뒤로가기를 수행하는지 확인한다.
  * **서버 동기화**: `UserDataManager`가 차감된 점수와 데이터의 로그가 서버 데이터베이스(`USER`, `DOPAMINE_LOG` 테이블)에 실시간으로 업데이트되는지 확인한다.

### 5.2.2. 모임방 및 웹 커뮤니티 연동 검증
**테스트 시나리오**:
  * **모임방 실시간 랭킹**: 도파민 점수가 차감될 경우 앱 내 모임방(`ROOM_MEMBER`)의 랭킹 및 자정 정산 결과가 실시간으로 변동되는지 확인한다.
  * **웹 커뮤니티 폰트 패널티**: 앱의 커뮤니티 탭을 통해 웹 플랫폼으로 진입하여 게시글/댓글을 작성할 때, 동기화된 당일 도파민 점수에 따른 글자 크기가 작게 혹은 크게 동적으로 변화되었는지 확인한다.
  * **아이템(확성기) 로직**: 인벤토리의 확성기 아이템 사용 시 생성된 게시글의 `is_ad` 상태가 TRUE로 변경되고, 피드 상단에 광고 형태로 정상적으로 노출되는지 검증한다.
